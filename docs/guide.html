1.3 Flux de travail typique

Analyse

Le prompt utilisateur est décomposé par LLM en tâches structurées (agent, action, params).

Routage

Les tâches sont ordonnancées : séquentiel, parallèle (≥2 agents différents), ou finalisation.

Exécution

Chaque agent exécute sa tâche avec retry (3 tentatives) et circuit breaker (3 échecs).

Vérification

Les résultats sont consolidés. Les actions HITL suspendent le workflow en attente d'approbation.

Finalisation

Résultat retourné, métriques enregistrées, session persistée.

HTTP Request
POST /api/v1/run
Content-Type: application/json
Authorization: Bearer <token>

{
  "prompt": "Crée un site e-commerce avec landing page et catalogue",
  "project_id": "mon-projet"
}

5.12 Nouveaux Modules v7.2.2

Sept nouveaux modules transforment AgentOS en système autonome avec raisonnement avancé, gestion mémoire GPU, et automatisation.

🌳 TreeOfThoughts — Raisonnement multi-branches

▼

Active le raisonnement arborescent quand la complexité d'une tâche dépasse 0.7. Jusqu'à 2 branches parallèles (TOT_MAX_BRANCHES=2 pour respecter le budget VRAM 12GB). Chaque branche explore une approche différente, et la meilleure solution est sélectionnée.

python
# Activation automatique dans l'orchestrateur
if task.complexity > 0.7:
    result = await tree_of_thoughts.solve(task)
else:
    result = await agent.execute(task)

🔗 GraphRAG — Graphe de connaissances

▼

Extraction d'entités et de relations après chaque session d'exécution. Stockage dans PostgreSQL avec format JSONB. Permet des requêtes cross-session pour un contexte enrichi. Appelé automatiquement par extract_and_store() en fin de session.

python
# Extraction automatique après chaque session
from app.memory.graph_rag import GraphRAG
graph = GraphRAG()
await graph.extract_and_store(session_id, text)

🎮 VRAM Manager — Gestion mémoire GPU (12GB)

▼

Gère la mémoire GPU pour les modèles Ollama. Bascule automatiquement entre modèles selon le work_type. Pour les tâches FAST, utilise qwen2.5:7b-instruct-q4_K_M (4.5GB). Ne jamais exécuter Computer Use et TreeOfThoughts simultanément.

Work TypeModèle OllamaVRAM
CODE_GEN / CODE_AGENTcodellama:13b~8GB
REASONINGmixtral:8x7b~10GB
FASTqwen2.5:7b-q4_K_M~4.5GB
CONTENT / GENERALllama3.2:3b~2GB

🔧 AutoCorrector — Correction automatique de code

▼

Wrapper autour de tous les appels sandbox DevAgent. Détecte les échecs d'exécution, analyse la cause racine via LLM, et réessaie avec du code corrigé. Zéro intervention manuelle nécessaire.

python
# Transparent pour l'agent
result = await auto_corrector.execute(
    agent=dev_agent,
    task=code_task,
    max_retries=3
)

⚡ WasmRunner — Sandbox WebAssembly

▼

Exécution sécurisée d'outils légers (<5ms) via WebAssembly. Sandbox isolé pour exécuter du code utilisateur sans risque. Utilisé par DevAgent pour les actions de scaffolding et de test.

python
from app.sandbox.wasm_runner import WasmRunner
runner = WasmRunner()
result = await runner.run_wasm(wasm_bytes, input_data)

🚌 AgentBus — Bus d'événements pour sous-agents

▼

Bus d'événements pour créer dynamiquement des sous-agents spécialisés. Permet la composition dynamique d'agents : création, communication, et retrait à chaud. Idéal pour les workflows complexes nécessitant des spécialistes temporaires.

python
# Spawn d'un sous-agent spécialisé
sub_agent = await agent_bus.spawn(
    agent_type="specialist",
    task=subtask
)
result = await sub_agent.run()

🖥️ ComputerUseTools — Automatisation desktop

▼

Outils d'automatisation du bureau : contrôle de la souris, du clavier, captures d'écran, presse-papier. Désactivé par défaut (COMPUTER_USE_ENABLED=true pour activer). Nécessite 12GB GPU libre et ne doit pas être utilisé simultanément avec TreeOfThoughts.

bash
# Activer dans .env
COMPUTER_USE_ENABLED=true
# Vérifier VRAM disponible
nvidia-smi --query-gpu=memory.free --format=csv
