Guide AgentOS v5.0

Plateforme d'orchestration multi-agent — Architecture, déploiement, API et développement

🏗️ 1. Architecture

AgentOS est une plateforme d'orchestration multi-agent basée sur FastAPI + LangGraph, avec persistance PostgreSQL/pgvector et cache Redis.

1.2 Composants clés

🌐

1.3 Flux de travail typique

Analyse

Le prompt utilisateur est décomposé par LLM en tâches structurées (agent, action, params).

Routage

Les tâches sont ordonnancées : séquentiel, parallèle (≥2 agents différents), ou finalisation.

Exécution

Chaque agent exécute sa tâche avec retry (3 tentatives) et circuit breaker (3 échecs).

Vérification

Les résultats sont consolidés. Les actions HITL suspendent le workflow en attente d'approbation.

Finalisation

Résultat retourné, métriques enregistrées, session persistée.

HTTP Request
POST /api/v1/run
Content-Type: application/json
Authorization: Bearer <token>

{
  "prompt": "Crée un site e-commerce avec landing page et catalogue",
  "project_id": "mon-projet"
}

5.3 Human-In-The-Loop

Les actions critiques suspendent le workflow en attente d'approbation humaine.

HTTP
# Approuver une action
POST /api/v1/hitl/approve
{ "approval_id": "uuid-xxx" }

# Rejeter
POST /api/v1/hitl/reject
{ "approval_id": "uuid-xxx", "reason": "Pas prêt" }

5.4 @Debugger Sub-Agent

Analyse d'erreurs avec suggestions de correction structurées.

HTTP
POST /api/v1/sub-agent/debug
{
  "error": "TypeError: expected string",
  "traceback": "Traceback (most recent call last)...",
  "context": { "file": "app/main.py", "line": 42 }
}

# → { root_cause, explanation, fix_suggestion, confidence }

5.5 Orchestrateur LangGraph

🔍 analyze_prompt
→
🔀 route_tasks
→
⚡ execute_agent
→
✅ check_results
→
🎯 finalize
↻ retry (×3) ⚠️ circuit breaker (×3)

Caractéristiques : Circuit Breaker (3 échecs → arrêt), Retry automatique (3 tentatives), Exécution parallèle (tâches indépendantes), HITL propagation (suspension sur actions critiques).

5.6 LLM Cache

Cache à 2 niveaux : L1 local (instantané) + Redis (persistant). Clé = SHA256(model + messages + temperature). TTL par défaut : 7 jours.

env
ENABLE_LLM_CACHE=true
LLM_CACHE_TTL=604800

5.7 Scheduler

Planification de tâches cron pour exécution automatique de prompts.

HTTP
POST /api/v1/scheduler/tasks
{
  "name": "Rapport hebdo",
  "cron": "0 9 * * 1",
  "prompt": "Génère le rapport d'activité",
  "channel": "slack"
}

6.3 Ajouter un agent

python
# 1. app/agents/mon_agent.py
from app.agents.base import BaseAgent

class MonAgent(BaseAgent):
    async def execute(self, task, session_id="", trace_id=""):
        return {"agent": "mon_agent", "success": True, "result": {}}

# 2. Dans orchestrator.py → __init__()
self.agents["mon_agent"] = MonAgent()
workflow.add_node("execute_mon_agent", self._execute_mon_agent)

6.4 Migrations

bash

# Appliquer
uv run alembic upgrade head

# Revenir en arrière
uv run alembic downgrade -1
